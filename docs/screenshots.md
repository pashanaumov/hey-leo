# Screenshot Plan

Keep screenshots consistent so Leo feels polished and curated.

## Setup

- Use the same VS Code window size for all captures.
- Use the same font, font size, icon theme, and sidebar layout.
- Keep tabs, sidebar, and status bar visible for at least one full UI screenshot.
- Prefer real code over toy snippets.

## Sample files

Use files from `screenshot-samples/` for repeatable captures:

- `screenshot-samples/leo-showcase.ts` — TypeScript hero with classes, literals, maps, async, and semantic tokens
- `screenshot-samples/leo-component.tsx` — React/TSX screenshot with JSX tags, props, hooks, callbacks, and strings
- `screenshot-samples/leo-config.json` — JSON screenshot with nested objects, booleans, arrays, and punctuation
- `screenshot-samples/leo-notes.md` — Markdown screenshot with headings, lists, quote, and code fence

- TypeScript

## Marketplace set

Use 4-6 final images:

- one flagship editor screenshot
- one Light screenshot
- one Dark Deep screenshot
- one full UI screenshot with sidebar/status bar
- one Dark vs Dark Deep vs Light comparison
- optional terminal or diff screenshot

## Naming
